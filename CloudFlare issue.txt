⚠️ Why the "Unexpected Error" happens in R2:

If refreshing the page does not remove the error, it means your company admin needs to quickly do one of two things on their end:

Billing Activation: Cloudflare R2 has a 100% free tier, but Cloudflare requires the Account Owner to have saved a billing card on file before the R2 service activates for the account.
Permission Role: Your invited role might not have "R2 Storage" permissions assigned to it.


Activate Cloudflare R2 Subscription:
Cloudflare offers a generous 100% Free Tier for R2 (up to 10GB of storage), but they strictly require a valid credit card/payment method to be on file to activate the R2 storage engine for the first time.

Assign Permissions to Your User Account:
The admin must go to Manage Account -> Members -> find your email -> and ensure you have been granted the "R2 Storage Admin" or "R2 Admin" role. (Without this, your dashboard cannot read or write to R2).

Cloudflare R2 Setup Guide for Company Administrators
This guide outlines the exact steps the Cloudflare account holder needs to take to set up the storage bucket and generate the credentials required for the FabricGuard backend system.

Prerequisites
A standard Cloudflare Account.
A valid Payment Method attached to the Cloudflare account (Cloudflare R2 requires this to activate the service, even though the Free Tier provides 10GB of storage, 1M Class A operations, and 10M Class B operations monthly at zero cost).
Step 1: Activate R2 Subscription
Log in to the Cloudflare Dashboard.
In the left-hand navigation sidebar, click on R2.
If R2 is not yet active, click the Purchase R2 or Subscribe button.
Verify the billing agreement ($0/month base subscription) and complete the activation.
Step 2: Create the Storage Bucket
On the R2 Dashboard, click the Create bucket button.
Under Bucket Name, enter a name using lowercase letters and hyphens (e.g., fabricguard-images).
Save this value for your configuration as: CLOUDFLARE_R2_BUCKET_NAME

Leave the Location settings as Automatic (it defaults to optimized performance).
Click Create bucket.
Step 3: Enable Public Access (For Browser Visibility)
By default, Cloudflare buckets are fully private. To allow the frontend application to display the uploaded fabric defect images, you must enable public read access.

Inside your newly created bucket, click on the Settings tab.
Scroll down to the Public Access section.
Under R2.dev Subdomain, click Connect or Allow Access.
Type confirm or click Allow when prompted.
Copy the generated public URL (it will look something like https://pub-xxxxxxxxxxxx.r2.dev).
Save this value for your configuration as: CLOUDFLARE_R2_PUBLIC_URL

Step 4: Generate API Tokens (For Backend Access)
To allow the Python backend server to securely upload images, you need an API keypair.

Navigate back to the main R2 landing page from the left-hand sidebar.
On the right-side of the page under Account details, click on the link that says Manage R2 API Tokens.
Click the Create API token button.
Set the following configurations:
Token name: FabricGuard-Backend-Access
Permissions: Select Edit (This permits the backend to write/upload files).
Specify bucket(s): Choose Apply to specific buckets only and select the bucket you created in Step 2.
Scroll to the bottom and click Create API Token.
CRITICAL: Copy the values presented on the next screen immediately. They will disappear forever once you leave the page:
Copy Access Key ID for: CLOUDFLARE_R2_ACCESS_KEY_ID
Copy Secret Access Key for: CLOUDFLARE_R2_SECRET_ACCESS_KEY
Step 5: Locate the Account ID
From any page on the Cloudflare R2 dashboard, look at the right-side sidebar under the Account details header.
Copy the long alpha-numeric string listed next to Account ID.
Save this value for your configuration as: CLOUDFLARE_ACCOUNT_ID
