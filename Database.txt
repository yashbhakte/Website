MongoDB


=  Yes, it can. MongoDB is a NoSQL database that stores data as JSON-like documents. For text data (like logins and history), it works perfectly. For images, MongoDB has a special feature called GridFS that allows you to store large files directly inside the database.


=  Honestly, no, it is not the best fit for your current setup, for two major reasons:

1. You would have to rewrite your existing backend code Right now, your backend uses a library called SQLAlchemy. This library is specifically designed for SQL databases (like PostgreSQL or SQLite). If you switch to MongoDB, you cannot use SQLAlchemy. We would have to completely delete your database.py file and rewrite all the database logic in your app.py using a different library (like PyMongo or Motor).






While MongoDB can do everything you are asking, it would require a massive rewrite of your backend, and serving images from it would eventually slow down your application.

The industry standard remains:

Database (PostgreSQL): For text, logins, and the URLs of the images.
Cloud Storage (Cloudinary / AWS S3): For the actual physical image files.
